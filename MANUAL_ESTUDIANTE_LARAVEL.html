📖 SECCIÓN 1: CONCEPTOS FUNDAMENTALES

1.1 ¿Qué es Laravel?

Laravel es un framework de PHP que nos ayuda a crear aplicaciones web de forma más rápida y organizada. Es como un "esqueleto" que ya tiene muchas funciones útiles predefinidas.

1.2 ¿Qué es MVC?

MVC significa Modelo-Vista-Controlador. Es una forma de organizar el código:
  • Modelo (M): Maneja los datos y la lógica de negocio
  • Vista (V): Lo que ve el usuario (páginas web)
  • Controlador (C): Conecta el modelo con la vista

1.3 ¿Qué es una Base de Datos?

Una base de datos es como una biblioteca digital donde guardamos toda la información de nuestra aplicación:
  • Información de usuarios
  • Datos de cursos
  • Calificaciones
  • Archivos

1.4 ¿Qué es AJAX?

AJAX permite que las páginas web se actualicen sin recargar completamente. En nuestro proyecto, se usa para cargar alumnos cuando seleccionas un curso.

2.1 Carpetas Principales

Gestion_Cursos/ ├── app/ # Código principal de la aplicación │ ├── Http/ │ │ ├── Controllers/ # Controladores (lógica de negocio) │ │ ├── Middleware/ # Filtros de seguridad │ │ └── Requests/ # Validación de formularios │ ├── Models/ # Modelos (interacción con base de datos) │ └── Services/ # Servicios adicionales ├── database/ │ ├── migrations/ # Estructura de la base de datos │ └── seeders/ # Datos de prueba ├── resources/ │ └── views/ # Páginas web (vistas) ├── routes/ │ └── web.php # Definición de URLs ├── storage/ # Archivos subidos └── public/ # Archivos públicos

2.2 ¿Qué hace cada carpeta?

📁 app/ - Código Principal

Controllers: Son como "jefes" que manejan las peticiones del usuario
Models: Son como "traductores" entre el código y la base de datos
Middleware: Son como "guardias de seguridad" que verifican permisos

📁 database/ - Base de Datos

Migrations: Son como "planos" que definen cómo crear las tablas
Seeders: Son como "plantillas" que llenan la base de datos con datos de prueba

📁 resources/views/ - Páginas Web

Vistas: Son las páginas HTML que ve el usuario
Layouts: Son como "moldes" que se reutilizan en varias páginas

📁 routes/ - URLs

web.php: Define qué URL va a qué función del controlador

🛣️ SECCIÓN 3: CÓMO FUNCIONAN LAS RUTAS EN LARAVEL

3.1 ¿Qué es una Ruta?

Una ruta es como una dirección que conecta una URL con una función específica. Es como decir: "Cuando alguien visite esta dirección, ejecuta esta función".

3.2 Ejemplo de Ruta

Route::get('/alumnos', [AlumnoController::class, 'index'])->name('alumnos.index');
Explicación:
  • Route::get = Cuando alguien visite la página (GET)
  • /alumnos = La dirección URL
  • AlumnoController::class = El controlador que maneja esto
  • 'index' = La función específica que se ejecuta
  • ->name('alumnos.index') = Un nombre para referenciar esta ruta

3.3 Tipos de Rutas

Tipo Uso Ejemplo
GET Mostrar información Ver lista de alumnos
POST Crear algo nuevo Crear un nuevo alumno
PUT/PATCH Actualizar algo existente Editar datos de un alumno
DELETE Eliminar algo Eliminar un alumno

3.4 Flujo de una Petición Web

1. Usuario hace clic en un enlace
↓
2. Laravel busca la ruta correspondiente
↓
3. Ejecuta el controlador indicado
↓
4. El controlador obtiene datos del modelo
↓
5. Se muestra la vista con los datos

🗄️ SECCIÓN 4: MODELOS Y BASE DE DATOS

4.1 ¿Qué es un Modelo?

Un modelo es una clase que representa una tabla de la base de datos. Es como un "traductor" que convierte los datos de la base de datos en objetos que podemos usar en PHP.

4.2 Ejemplo de Modelo

class Alumno extends Model { // Relación con inscripciones public function inscripciones() { return $this->hasMany(Inscripcion::class); } // Scope para alumnos activos public function scopeActivos($query) { return $query->where('activo', true); } }

4.3 ¿Qué son las Migraciones?

Las migraciones son archivos que definen la estructura de las tablas de la base de datos. Son como "planos de construcción" para crear las tablas.

4.4 Ejemplo de Migración

public function up() { Schema::create('alumnos', function (Blueprint $table) { $table->id(); $table->string('nombre'); $table->string('apellido'); $table->string('email')->unique(); $table->boolean('activo')->default(true); $table->timestamps(); }); }

4.5 ¿Qué son los Seeders?

Los seeders son archivos que llenan la base de datos con datos de prueba. Son útiles para probar la aplicación sin tener que crear datos manualmente.

🎮 SECCIÓN 5: CONTROLADORES

5.1 ¿Qué es un Controlador?

Un controlador es una clase que maneja las peticiones del usuario. Es como un "jefe" que decide qué hacer cuando alguien visita una página.

5.2 Métodos Principales de un Controlador

Método Propósito Ejemplo
index() Mostrar lista de elementos Lista de todos los alumnos
create() Mostrar formulario de creación Formulario para crear alumno
store() Guardar nuevo elemento Guardar datos del nuevo alumno
show() Mostrar elemento específico Ver detalles de un alumno
edit() Mostrar formulario de edición Formulario para editar alumno
update() Actualizar elemento Guardar cambios del alumno
destroy() Eliminar elemento Eliminar un alumno

5.3 Ejemplo de Controlador

class AlumnoController extends Controller { public function index() { $alumnos = Alumno::activos()->paginate(10); return view('alumnos.index', compact('alumnos')); } public function store(Request $request) { $request->validate([ 'nombre' => 'required|string|max:255', 'email' => 'required|email|unique:alumnos' ]); Alumno::create($request->all()); return redirect()->route('alumnos.index'); } }

🎨 SECCIÓN 6: VISTAS Y TEMPLATE ENGINE BLADE

6.1 ¿Qué es Blade?

Blade es el motor de plantillas de Laravel. Permite crear páginas HTML dinámicas mezclando HTML con código PHP de forma segura y fácil.

6.2 Sintaxis Básica de Blade

{{-- Mostrar variable --}} {{ $alumno->nombre }} {{-- Condicionales --}} @if($alumno->activo) Activo @else Inactivo @endif {{-- Bucles --}} @foreach($alumnos as $alumno) {{ $alumno->nombre }} {{ $alumno->email }} @endforeach

6.3 Layouts y Secciones

💼 SECCIÓN 7: LÓGICA DE NEGOCIO DEL PROYECTO

7.1 ¿Qué es la Lógica de Negocio?

La lógica de negocio son las reglas que definen cómo debe funcionar la aplicación. Por ejemplo: "Un docente solo puede tener máximo 3 cursos activos".

7.2 Reglas de Negocio Implementadas

📚 Gestión de Cursos

Regla: Un docente solo puede tener máximo 3 cursos activos
Implementación: Validación en el controlador de cursos
Ubicación: CursoController.php

👥 Gestión de Alumnos

Regla: Un alumno solo puede ser aprobado si tiene al menos 75% de asistencia
Implementación: Validación en InscripcionController
Ubicación: InscripcionController.php - método cambiarEstado()

📝 Gestión de Evaluaciones

Regla: Solo se pueden crear evaluaciones para alumnos inscritos activamente
Implementación: Filtrado en AJAX y validación en controlador
Ubicación: EvaluacionController.php - método getAlumnosByCurso()

7.3 Ejemplo de Validación de Negocio

// En InscripcionController.php public function cambiarEstado(Inscripcion $inscripcion, $nuevoEstado) { if ($nuevoEstado === 'aprobado') { $porcentajeAsistencia = ($inscripcion->asistencias / 20) * 100; if ($porcentajeAsistencia < 75) { return redirect()->back()->with('error', 'No se puede aprobar. Se requiere al menos 75% de asistencia.'); } } $inscripcion->update(['estado' => $nuevoEstado]); return redirect()->back()->with('success', 'Estado actualizado.'); }

🔐 SECCIÓN 8: SISTEMA DE AUTENTICACIÓN Y ROLES

8.1 ¿Qué es la Autenticación?

La autenticación es el proceso de verificar quién es el usuario. Es como mostrar tu identificación para entrar a un lugar.

8.2 Roles del Sistema

Rol Permisos Funcionalidades
Admin Acceso completo Todo el sistema
Coordinador Gestión de alumnos e inscripciones Alumnos, inscripciones, evaluaciones
Docente Solo sus cursos Sus cursos, alumnos, evaluaciones, archivos

8.3 Middleware de Autenticación

// En routes/web.php Route::middleware(['auth', 'role.admin'])->group(function () { Route::get('/admin/dashboard', [AdminController::class, 'dashboard']); }); // En controladores public function __construct() { $this->middleware('auth'); $this->middleware('role.docente'); }

🔄 SECCIÓN 9: AJAX Y INTERACTIVIDAD

9.1 ¿Qué es AJAX?

AJAX permite actualizar partes de una página web sin recargar toda la página. Mejora la experiencia del usuario haciéndola más fluida.

9.2 AJAX en el Proyecto

Funcionalidad: Cargar alumnos por curso
Cuándo se usa: Al crear una evaluación
Beneficio: Solo muestra alumnos relevantes

9.3 Ejemplo de Código AJAX

// JavaScript en la vista fetch(`/evaluaciones/alumnos-by-curso?curso_id=${cursoId}`, { method: 'GET', headers: { 'Accept': 'application/json', 'X-CSRF-TOKEN': token } }) .then(response => response.json()) .then(alumnos => { // Actualizar dropdown con los alumnos }); // Controlador que responde public function getAlumnosByCurso(Request $request) { $alumnos = Alumno::whereHas('inscripciones', function($query) use ($request) { $query->where('curso_id', $request->curso_id) ->where('estado', 'activo'); })->get(); return response()->json($alumnos); }

🔧 SECCIÓN 10: PROBLEMAS RESUELTOS EN EL PROYECTO

10.1 Error 404 en AJAX

Problema: La ruta AJAX devolvía 404 (página no encontrada)
Causa: Las rutas específicas estaban después de las rutas resource
Solución: Reordenar las rutas en web.php

10.2 Inconsistencias en Base de Datos

Problema: Datos de prueba no cumplían las reglas de negocio
Causa: Docentes con más de 3 cursos, alumnos sin inscripciones
Solución: Crear migración para corregir datos

10.3 Problemas de Permisos

Problema: Coordinadores veían opciones que no podían usar
Causa: Menú no estaba condicionado por rol
Solución: Agregar condiciones en las vistas
